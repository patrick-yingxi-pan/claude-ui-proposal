## Standing assumptions (set at kickoff)

These frame every decision below; called out so the end review can challenge the frame,
not just the leaves:

- **Target = both deployments** (desktop sidecar + multi-tenant web), one shared contract.
- **Design, not build** â€” engineering-design altitude; the prototype is not converted.
- **Preserve the prototype's locked invariants** where they still apply in production
  (one door to the backend, framework-free contract, consent-gated irreversible effects,
  the model as the generation seam) unless a decision explicitly revisits one (flagged).
- **Cloud vendor-neutral by default** â€” name a concrete technology only as an example,
  and prefer portable abstractions; any hard dependency is flagged `âš  confirm`.
